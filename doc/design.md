# 수신호 기반 공정 기계 안내 시뮬레이터 (mocap-sample)

## 프로젝트 개요

OpenCV와 MediaPipe를 이용한 모션/포즈 인식으로 **산업 현장 수신호(hand signal)** 를 감지하고,
머신러닝 학습을 통해 카메라에 인식되는 수신호를 분류하는 프로젝트.

**최종 목표:** 웹캠 앞에서 작업자가 수신호를 하면 Gazebo 시뮬레이션의 공정 기계(로봇)가
이를 인식·예측하여 대응 행동을 취하는 것.
예: 기계가 작업자 쪽으로 접근하다가 '멈춰' 수신호 → 정지, '천천히' → 감속, '오지마' → 후진.

**이 저장소의 범위는 수신호 인식(데이터 수집 → 학습 → 실시간 추론)까지.**
ROS 2 / Gazebo 기계 제어는 별도 프로젝트에서 다루며, 이 저장소는 분류 결과를
외부로 전달하는 것까지만 책임진다.

## 시스템 파이프라인

```
카메라 입력 (OpenCV)
   ↓
손/포즈 랜드마크 추출 (MediaPipe HandLandmarker)
   ↓
랜드마크 시퀀스 전처리 (정규화, 시퀀스 버퍼링)
   ↓
수신호 분류 모델 추론 (학습된 ML 모델)
   ↓
분류 결과 출력/전달           ← 이 저장소의 책임은 여기까지
   ↓
(별도 프로젝트) ROS 2 → Gazebo 공정 기계 행동 실행 (정지/감속/후진 등)
```

## 기술 스택

| 구성 요소 | 기술 |
|---|---|
| 영상 입력/처리 | OpenCV |
| 랜드마크 추출 | MediaPipe HandLandmarker + PoseLandmarker (Tasks API) — 양손 + 상체 관절 |
| 수신호 분류 모델 | LSTM/GRU 계열 시퀀스 분류 |
| 언어 | Python 3.12 (의존성은 pyproject.toml에 버전 고정) |

(로봇 미들웨어 ROS 2, 시뮬레이터 Gazebo는 별도 프로젝트 담당)

## 모듈 구성

```
mocap-sample/
├── doc/                    # 문서
├── asset/                  # 수집한 수신호 랜드마크 데이터셋 (.npy)
├── models/                 # MediaPipe 모델 + 학습된 분류 모델
└── src/
    ├── capture/            # 카메라 캡처 + MediaPipe 랜드마크 추출
    ├── dataset/            # 데이터 수집 스크립트 (수신호별 라벨링·저장)
    ├── training/           # 모델 학습 스크립트
    └── inference/          # 실시간 수신호 인식 (추론) + 결과 외부 전달 인터페이스
```

## 개발 단계

1. **랜드마크 추출 검증** — 완료 (`src/capture/landmark_viewer.py`)
2. **데이터 수집** — 수신호 클래스별 랜드마크 시퀀스를 라벨과 함께 저장 (`src/dataset/collect.py`)
3. **모델 학습** — 랜드마크 시퀀스로 분류 모델 학습
4. **실시간 추론** — 카메라 입력에서 실시간으로 수신호를 분류하고 결과 출력
5. **결과 전달 인터페이스** — 분류 결과를 기계 시뮬레이터 프로젝트(별도 저장소)가 받을 수 있는 형태로 전달 (방식 미정: 소켓, rosbridge 등)

## 주요 설계 결정 사항

### 결정됨

- **동적 수신호(시퀀스)** — 시퀀스당 30프레임, LSTM/GRU 계열 모델 사용
- **특징 = 양손 + 상체 포즈 150차원** (`src/capture/extractor.py`)
  - 왼손 63 + 오른손 63 (21랜드마크 × xyz) + 상체 8관절(어깨·팔꿈치·손목·골반) × xyz = 24
  - 포즈를 넣은 이유: 산업 수신호는 팔 궤적이 본체이고, 목장갑 등으로 손 랜드마크가
    불안정한 환경에서도 포즈는 몸 전체 스케일이라 강인하다. 골반 포함으로 상반신 회전 표현.
- **데이터 형식**: `asset/<라벨>/<타임스탬프>.npy`, shape (30, 150) — 감지 안 된 부분은 0
- **학습 데이터는 자체 수집** — 수신호는 프로젝트별 정의가 달라 자체 수집이 정석.
  `src/dataset/collect.py`로 웹캠에서 직접 수집
- **안전 기본값** — 신호가 불확실하면(신뢰도 낮음) 기계는 '정지'로 동작해야 한다

- **클래스 목록은 고정하지 않는다** — `asset/`의 라벨 폴더가 곧 클래스다.
  학습 스크립트는 폴더를 스캔해 클래스를 동적으로 구성하고, 라벨 목록을 모델과 함께 저장한다.
  추론은 저장된 라벨 목록을 읽는다. 코드에 클래스 이름을 하드코딩하지 말 것.
- **미학습 동작 처리** — 분류 모델은 학습한 클래스만 예측 가능하므로,
  신뢰도 임계값 미만이면 "인식 불가"로 처리하고 안전 기본값(정지)을 따른다.

### 미정

- 분류 결과를 기계 프로젝트로 전달하는 방식 (소켓, rosbridge 웹소켓 등)

## 환경 참고

- 개발 머신: **Windows / macOS 둘 다 지원** — MediaPipe, OpenCV, 모델 학습·추론 모두 양쪽에서 네이티브 실행
- 모든 코드는 크로스 플랫폼으로 작성 (경로는 `pathlib`, 카메라 백엔드는 OS별 분기, 셸 스크립트 대신 Python 스크립트)
- 웹캠을 사용하는 코드는 Docker에 넣지 않는다 (Windows/macOS Docker 모두 카메라 접근 불가)
- 의존성 설치: 프로젝트 루트의 `.venv` (Python 3.12) 에 `pip install -e .` — OS별 생성 방법은 `doc/AGENT.md` 셋업 참고
