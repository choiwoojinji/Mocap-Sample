# 수화 인식 프로젝트 (mocap-sample)

## 프로젝트 개요

OpenCV와 MediaPipe를 이용한 모션/포즈 인식으로 수화(수어) 동작을 감지하고,
머신러닝 학습을 통해 카메라에 인식되는 수화 동작을 분류하는 프로젝트.

**이 저장소의 범위는 수화 인식(데이터 수집 → 학습 → 실시간 추론)까지.**
최종 목표는 웹캠 앞에서 사용자가 수화를 하면 Gazebo 시뮬레이션의 로봇이 이를
인식·예측하여 대응 행동을 취하는 것이지만, ROS 2 / Gazebo 로봇 제어는
**별도 프로젝트**에서 다룬다. 이 저장소는 분류 결과를 외부로 전달하는 것까지만 책임진다.

## 시스템 파이프라인

```
카메라 입력 (OpenCV)
   ↓
손/포즈 랜드마크 추출 (MediaPipe Hands / Holistic)
   ↓
랜드마크 시퀀스 전처리 (정규화, 시퀀스 버퍼링)
   ↓
수화 분류 모델 추론 (학습된 ML 모델)
   ↓
분류 결과 출력/전달           ← 이 저장소의 책임은 여기까지
   ↓
(별도 프로젝트) ROS 2 → Gazebo 로봇 행동 실행
```

## 기술 스택

| 구성 요소 | 기술 |
|---|---|
| 영상 입력/처리 | OpenCV |
| 손·포즈 랜드마크 추출 | MediaPipe (Hands 또는 Holistic) |
| 수화 분류 모델 | TensorFlow/Keras LSTM (시퀀스 동작) 또는 scikit-learn (정적 수화) |
| 언어 | Python 3.12 (의존성은 pyproject.toml에 버전 고정) |

(로봇 미들웨어 ROS 2, 시뮬레이터 Gazebo는 별도 프로젝트 담당)

## 모듈 구성 (계획)

```
mocap-sample/
├── doc/                    # 문서
├── asset/                  # 수집한 수화 랜드마크 데이터셋 (.npy)
├── models/                 # 학습된 모델 파일
└── src/
    ├── capture/            # 카메라 캡처 + MediaPipe 랜드마크 추출
    ├── dataset/            # 데이터 수집 스크립트 (수화별 라벨링·저장)
    ├── training/           # 모델 학습 스크립트
    └── inference/          # 실시간 수화 인식 (추론) + 결과 외부 전달 인터페이스
```

## 개발 단계

1. **랜드마크 추출 검증** — OpenCV로 웹캠 영상을 받아 MediaPipe로 손/포즈 랜드마크를 실시간 시각화
2. **데이터 수집** — 인식할 수화 단어(클래스)를 정하고, 수화별 랜드마크 시퀀스를 라벨과 함께 저장
3. **모델 학습** — 랜드마크 시퀀스로 분류 모델 학습 (동적 수화는 LSTM/GRU, 정적 수화는 SVM/RandomForest도 가능)
4. **실시간 추론** — 카메라 입력에서 실시간으로 수화를 분류하고 결과 출력
5. **결과 전달 인터페이스** — 분류 결과를 로봇 프로젝트(별도 저장소)가 받을 수 있는 형태로 전달 (방식 미정: 소켓, rosbridge 등)

## 주요 설계 결정 사항

### 결정됨

- **수화 단어(클래스): 예 / 아니요 / 안녕하세요** — 3개로 시작, 추후 확장
- **동적 수화(시퀀스)** — 시퀀스당 30프레임, LSTM/GRU 계열 모델 사용
- **데이터 형식**: `asset/<라벨>/<타임스탬프>.npy`, shape (30, 126)
  - 126 = 두 손 × 21 랜드마크 × (x, y, z), 안 보이는 손은 0
- **학습 데이터는 자체 수집** — `src/dataset/collect.py`로 웹캠에서 직접 수집
  (외부 데이터셋(AI Hub 수어 영상)은 추후 일반화가 필요할 때 랜드마크 재추출해서 증강용으로 검토)

### 미정

- 분류 결과를 로봇 프로젝트로 전달하는 방식 (소켓, rosbridge 웹소켓 등)

## 환경 참고

- 개발 머신: **Windows / macOS 둘 다 지원** — MediaPipe, OpenCV, 모델 학습·추론 모두 양쪽에서 네이티브 실행
- 모든 코드는 크로스 플랫폼으로 작성 (경로는 `pathlib`, 카메라 백엔드는 OS별 분기, 셸 스크립트 대신 Python 스크립트)
- 웹캠을 사용하는 코드는 Docker에 넣지 않는다 (Windows/macOS Docker 모두 카메라 접근 불가)
- 의존성 설치: 프로젝트 루트의 `.venv` (Python 3.12) 에 `pip install -e .` — OS별 생성 방법은 `doc/AGENT.md` 셋업 참고
