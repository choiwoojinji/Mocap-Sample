# 피지컬 AI 기법 노트 — 센서 처리·안전 제어·경로 계획

피지컬 AI 실습 자료(2026-07-09 학습)에서 이 프로젝트에 적용 가치가 있는 기법을
분석·정리한 문서. 각 절은 "원리 → 왜 중요한가 → 우리 시스템 적용" 순서.

## 1. 스파이크 노이즈와 시계열 필터링

**문제의 본질**: 실환경 센서는 파리·먼지·조명 반사·전자파 때문에 순간적으로
말이 안 되는 값을 뱉는다 (예: 2m에 있던 사람이 0.5초 만에 0.1m로 "순간이동").
이 스파이크를 그대로 믿으면 로봇이 하루 수십 번 불필요한 긴급 정지를 반복하고,
결국 현장에서 시스템을 꺼버린다 — **노이즈 방어는 안전 문제이자 가용성 문제다.**
(우리가 ODD 논의에서 도달한 "벌레 한 마리가 공장을 세우면 안 된다"와 동일한 결론)

**기법 계층**:
| 기법 | 원리 | 특성 |
|---|---|---|
| 단순 이동 평균 (SMA) | 최근 N개 값의 평균 (`deque(maxlen=N)` 슬라이딩 버퍼) | 구현 단순, 반응 지연 있음 |
| 지수 평활 (EMA) | 최근 값에 지수적으로 큰 가중치 | 지연 적고 노이즈 억제 — 실시간 제어에 적합 |
| 칼만 필터 | 센서 오차 확률 모델까지 반영해 상태 추정 | 자율주행 표준, 구현 비용 높음 |

**판단 로직과 필터의 분리가 핵심**: 판단(정지/감속)은 필터를 통과한 값으로만 한다.
날값 → 필터 → 판단의 3단 구조.

**우리 시스템 적용**:
- 현재 안정화 필터("신뢰도 0.8 + 연속 5회 일치")는 이진 판정 방식의 스파이크 방어다.
  **클래스 확률 벡터에 EMA를 걸면** 연속성이 생겨 순간 튐에 더 부드럽게 강해지고,
  백로그의 히스테리시스(직전 신호 N초 유지)와 자연스럽게 결합된다.
- 랜드마크 좌표 자체의 EMA 평활도 가능하지만, 빠른 신호 동작의 반응성을 깎는
  트레이드오프가 있어 저조도 실측에서 떨림이 문제로 확인될 때만 도입.

## 2. 안전 구역 기반 점진 제어 (인간-로봇 협업 HRC)

**원리**: "가동 아니면 정지"의 이진 제어가 아니라, **거리 구역 × 대상 확신도**의
2차원 판단으로 단계적으로 감속한다:

```
안전 구역 (>2m)      → 정상 속도 100%
경고 구역 (1~2m)     → 사람 확률 ≥ 0.7이면 감속 50%
위험 구역 (<1m)      → 사람 확률 ≥ 0.7이면 긴급 정지
```

두 가지 통찰이 들어 있다:
1. **점진 감속이 안전과 가용성의 절충점** — 사람이 접근 중이라고 바로 세우지 않고
   감속으로 대응하면, 생산은 유지하면서 사고 에너지는 줄어든다.
2. **확신도 게이트** — "사람일 확률 70% 이상일 때만" 안전 프로토콜을 발동해
   물체 오인으로 인한 불필요 정지를 걸러낸다. (우리의 신뢰도 임계값 0.8과 같은 사상)

**우리 시스템 적용**:
- 지금 기계 제어는 수신호로만 속도를 정한다. 여기에 **작업자 거리 구역을 곱하면**
  "come 신호여도 1m 이내로 접근하면 강제 감속/정지" 같은 계층 안전이 된다 —
  신호(사람의 의도)와 구역(물리적 안전)을 독립된 방어선으로 두는 구조.
- 시야 자가 진단(design.md C절)의 DEGRADED → 감속 정책과 같은 사상이라,
  구현 시 하나의 "속도 결정 함수"로 통합 가능: `속도 = min(신호 속도, 구역 상한, 진단 상한)`.

## 3. 인공 포텐셜 필드(APF) 경로 계획

**원리**: 목적지는 인력(끌어당김), 장애물은 반경 안에서 거리에 반비례하는
척력(밀어냄)으로 모델링하고, 매 스텝 두 힘의 벡터 합 방향으로 이동한다.
"3m 가서 우회전" 같은 규칙을 코딩하지 않아도 **매력(+)과 위험(-) 수치 두 개만으로
회피 곡선이 창발**하는 것이 요점.

**한계도 알아둘 것**: 인력과 척력이 상쇄되는 지점(국소 최솟값)에 로봇이 갇힐 수 있어,
복잡한 환경에서는 전역 계획(A* 등)과 병용하는 게 정석이다.

**우리 시스템 적용**:
- 시뮬레이터 월드 확장 단계에서 기계의 이동 방식으로 채택 후보: come 신호에
  작업자에게 접근할 때 적재물·장비를 APF로 우회.
- **작업자 자신을 약한 척력원으로 설정**하면 "부르면 다가오되 일정 거리 안으로는
  들어오지 않는" 행동이 규칙 없이 자연 구현된다 — 2절의 위험 구역과 결합되는 지점.

## 4. 실환경 센서 데이터의 3대 결함과 전처리

전처리가 피지컬 AI 엔지니어링 시간의 70%를 차지한다는 것이 실무 정설이며,
결함은 세 유형으로 정리된다:

| 결함 | 원인 | 표준 처방 |
|---|---|---|
| 스파이크 노이즈 | 진동·반사·먼지·전자파 | 1절의 필터링 |
| 샘플링 주파수 불일치 | 센서마다 다른 출력 주기 (라이다 100Hz vs 카메라 30Hz) | 시간 동기화 |
| 결측치 | 통신 두절·가림 | 보간(interpolation) |

**우리 시스템 대조**: 이미 같은 범주의 방어를 여럿 갖췄다 — 품질 필터(수집),
손 소실 드롭아웃 증강(학습), 슬라이딩 윈도우·안정화 필터(추론), 하트비트(전달).
주파수 불일치는 현재 단일 카메라 + 1Hz 하트비트 구조라 문제없지만,
다중 센서(IR 카메라, 거리 센서)를 도입하는 순간 시간 동기화가 첫 과제가 된다.
